Mental Health Feature at Internship

Designed a mood-based feature on their website using Laravel, PHP, SQL and JavaScript.

  • Worked with Docker (containers, volumes) and TablePlus for SQL database management.
  • Used Git (branching, rebasing) and collaborated on feature development through pull requests.
  • Built and tested a mental health feature using PHP, Laravel, and RESTful APIs, with SEO and accessibility considerations.
  • Gained exposure to CI/CD pipelines, Kubernetes, and Infrastructure as Code.
  • Implemented secure API authentication using JWTs, refresh tokens, rate limiting, and IP blocking.
  • Used Supabase and PostgreSQL (RLS), plus Redis and Postman for backend testing.
  • Applied unit testing to ensure stability during development changes.

During my internship at TruTrade Group, I developed a mental health and mood-tracking feature for the company's web platform. The challenge was to design a system that could help users monitor their wellbeing and receive meaningful insights, but the business requirements were initially vague and ambiguous. There was no clear definition of what “improving mental health engagement” meant or how success would be measured. This ambiguity forced me to rely heavily on data and feedback to drive decisions and validate my own design choices, while also integrating DevOps practices to ensure the feature could be deployed and iterated on reliably.

I began by building a daily check-in feature for the users using HTML, CSS, JavaScript and PHP. I collected the user interaction data to understand the wellbeing of the users. The dataset included both quantitative metrics such as daily check-in frequency and average session duration, and qualitative feedback about the user experience. My goal was to use this data to shape the feature's design, focusing on how to increase participation and retention over time. Alongside this, I containerised parts of the workflow using Docker to create consistent development environments and streamline testing.

To analyse the data, I used Python libraries such as Pandas, NumPy and Matplotlib for pattern recognition and to visualise charts for each specific user. In order to persist the user data, I used TablePlus and SQL for structured database queries, and later introduced dashboards to track engagement metrics over the development cycle. As the platform's infrastructure grew, I gained exposure to CI/CD pipelines and Kubernetes, learning how container orchestration and automated deployments support scalable and maintainable feature rollouts.

Based on this, I introduced data-driven prompts, for example showing motivational quotes or tailored wellness messages based on a user's emotional state. Further down the line, I designed five interactive mood states: happy, neutral, sad, bored, and anxious, each offering a tailored experience such as mini-games, break time, journalling and more. Each state had unique logic for updating user points and tracking engagement data. This ensured that user retention remained strong throughout the feature, with DevOps tools helping me continuously test and release updates with minimal disruption.

As the feature developed, the analytics revealed clear engagement spikes in the happy and bored states; users appreciated activities that were immediate and interactive. I used this data to iterate the design and prioritise features that maintained consistent user participation. After launch, the feature increased daily check-in activity by over 25% and improved session retention across the site.

This experience taught me that ambiguity can drive innovation when guided by data. By breaking the project into smaller goals, testing hypotheses, and validating each stage with real user behaviour, I gained confidence in both technical decision-making and infrastructure awareness. It strengthened my communication skills, as I presented to both technical and non-technical stakeholders.

Data is fundamental to Technology Management because it converts uncertainty into clarity and supports decisions that balance business goals with user experience. I believe this mindset, supported by DevOps practices and an understanding of scalable infrastructure, would allow me to make a positive contribution from day one.

  • Implemented a comprehensive search cell to filter music records based on artist, title, medium, and genre, displaying detailed information and availability
  • Demonstrated proficiency in using Python to manage data structures, file handling and implemented inventory pruning using Matplotlib

I built a music store rental management system in Python as part of my coursework at Loughborough University. The project aimed to simulate a real-world music store rental system, allowing users to rent and manage their music collections.
